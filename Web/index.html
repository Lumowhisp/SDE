📐 Foundations

Why Sliding Window?

The sliding window maintains a persistent state for a contiguous segment, updating it incrementally as boundaries advance — eliminating redundant computation.

🐢

Brute Force

O(n²) or O(n³)

Recalculates every subarray from scratch. For each starting position, iterates through all possible ending positions. High redundancy — overlapping segments are re-processed entirely.

⚡

Sliding Window

O(n)

Incremental update of persistent state. Each element is visited at most twice (once by each pointer). The transition from range [i, j] to [i+1, j+1] involves only two operations.

Model Strategy Time Redundancy
Brute Force Recalculate every subarray O(n²) 🔴 High
Sliding Window Incremental state update O(n) 🟢 Low
Two-Pointer Independent movement O(n) 🟡 Variable

01

Contiguous Aggregates (Fixed Sum)

Fixed Window Easy

The most fundamental fixed-size pattern — calculate a numeric property (sum, average, max) across all subarrays of length k.

The core idea: "prime" the window by calculating the initial aggregate for the first k elements, then slide by adding the incoming element and subtracting the outgoing element.

💡 Key Formula

sum[i] = sum[i-1] - arr[i-k] + arr[i] — each transition is O(1), making the total O(n).

▶ Maximum Sum Subarray of size K
C++ — Max Sum Subarray
int maxSumSubarray(vector<int>& arr, int k) {
    int n = arr.size();
    // Prime the window
    int windowSum = accumulate(arr.begin(), arr.begin() + k, 0);
    int maxSum = windowSum;

    // Slide the window
    for (int i = k; i < n; i++) {
        windowSum += arr[i] - arr[i - k];  // add incoming, remove outgoing
        maxSum = max(maxSum, windowSum);
    }
    return maxSum;
}
02

Permutation & Anagram Detection

Fixed Window Medium

Detect if any window of size |P| in string S is a permutation (anagram) of pattern P. The state is a frequency distribution rather than a scalar.

Use two frequency arrays of size 26. As the window slides, increment the entering character's count and decrement the leaving character's count. A "matches" counter tracks how many of 26 characters have equal frequencies.

💡 Key Insight

When matches == 26, all character frequencies are equal — the window is an anagram! This gives O(1) comparison after O(|P|) setup.

▶ Find Anagram in String
C++ — Permutation in String
vector<int> findAnagrams(string s, string p) {
    vector<int> result;
    if (s.size() < p.size()) return result;

    int pCount[26] = {}, sCount[26] = {};
    for (char c : p) pCount[c - 'a']++;

    int matches = 0;
    for (int i = 0; i < 26; i++)
        if (pCount[i] == 0) matches++;  // both zero = match

    for (int i = 0; i < s.size(); i++) {
        // Add right char
        int idx = s[i] - 'a';
        sCount[idx]++;
        if (sCount[idx] == pCount[idx]) matches++;
        else if (sCount[idx] == pCount[idx] + 1) matches--;

        // Remove left char (when window exceeds |P|)
        if (i >= p.size()) {
            idx = s[i - p.size()] - 'a';
            sCount[idx]--;
            if (sCount[idx] == pCount[idx]) matches++;
            else if (sCount[idx] == pCount[idx] - 1) matches--;
        }

        if (matches == 26)
            result.push_back(i - p.size() + 1);
    }
    return result;
}
03

Longest Subarray with K Distinct

Variable Window Medium

Find the longest substring/subarray with at most K distinct elements. The window expands freely and contracts only when the constraint is violated.

The state is tracked with std::unordered_map for character frequency counts. When map.size() > K, contract from the left. Crucially: erase keys with zero frequency to correctly maintain size().

💡 Expansion-Contraction Rhythm

Expand → add element, Validate → check constraint, Contract → shrink until valid, Report → update max length.

▶ Longest with K Distinct Characters
C++ — Longest Substring with K Distinct
int longestWithKDistinct(string s, int k) {
    unordered_map<char, int> freq;
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.size(); right++) {
        freq[s[right]]++;  // Expand

        // Contract while constraint violated
        while (freq.size() > k) {
            freq[s[left]]--;
            if (freq[s[left]] == 0)
                freq.erase(s[left]);  // crucial!
            left++;
        }

        maxLen = max(maxLen, right - left + 1);  // Report
    }
    return maxLen;
}
04

Shortest Subarray with Target Sum

Variable Window Medium

Find the shortest subarray whose sum is ≥ target S. The logic is inverted from the "longest" pattern — once the sum reaches the target, contract as much as possible.

The critical difference: for longest problems, update result after expansion; for shortest problems, update result during the contraction phase as long as the window remains valid.

💡 Longest vs Shortest

Longest → update outside the while loop. Shortest → update inside the while loop. This is the #1 mistake in interviews!

▶ Minimum Size Subarray Sum ≥ S
C++ — Minimum Size Subarray Sum
int minSubArrayLen(int target, vector<int>& nums) {
    int left = 0, sum = 0;
    int minLen = INT_MAX;

    for (int right = 0; right < nums.size(); right++) {
        sum += nums[right];  // Expand

        // Contract while valid — update INSIDE the loop!
        while (sum >= target) {
            minLen = min(minLen, right - left + 1);
            sum -= nums[left];
            left++;
        }
    }
    return minLen == INT_MAX ? 0 : minLen;
}
05

Monotonic Deque — Sliding Window Maximum

Fixed Window Hard

Find the maximum element in every window of size K. A naive heap approach gives O(n log n), but a monotonic deque achieves O(n).

The deque stores indices in decreasing order of values. Three key operations per step:

1. Remove front if outside window (deque.front() ≤ i - k)
2. Pop back while arr[back] ≤ arr[i] (they can never be the max)
3. Push current index — deque.front() is always the max

💡 Why It Works

An older, smaller element can never be the maximum of any future window — a newer, larger element dominates it. So we discard it.

▶ Sliding Window Maximum (LC 239)
Deque:
C++ — Sliding Window Maximum
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;  // stores indices, decreasing order of values
    vector<int> result;

    for (int i = 0; i < nums.size(); i++) {
        // 1. Remove elements outside the window
        if (!dq.empty() && dq.front() <= i - k)
            dq.pop_front();

        // 2. Maintain monotonicity — pop smaller elements
        while (!dq.empty() && nums[dq.back()] <= nums[i])
            dq.pop_back();

        dq.push_back(i);

        // 3. Record result once window is full
        if (i >= k - 1)
            result.push_back(nums[dq.front()]);
    }
    return result;
}
06

Sliding Window Median (Two Heaps)

Fixed Window Hard

Maintain the median of a moving window using a Two-Heap strategy:

• Max-Heap — contains the smaller half of window elements
• Min-Heap — contains the larger half of window elements

The median is the top of the larger heap (odd size) or average of both tops. A lazy deletion mechanism handles removals: track pending removals in a hash map and only delete when they reach a heap top.

💡 Lazy Deletion

Instead of searching O(n) to remove an element from a heap, mark it as "pending" and remove only when it reaches the top during rebalancing.

C++ — Sliding Window Median
vector<double> medianSlidingWindow(vector<int>& nums, int k) {
    multiset<int> lo, hi;  // lo = max-heap equiv, hi = min-heap equiv
    vector<double> result;

    for (int i = 0; i < nums.size(); i++) {
        // Insert into appropriate half
        if (lo.empty() || nums[i] <= *lo.rbegin())
            lo.insert(nums[i]);
        else
            hi.insert(nums[i]);

        // Rebalance: lo.size() == hi.size() or lo.size() == hi.size() + 1
        while (lo.size() > hi.size() + 1) {
            hi.insert(*lo.rbegin());
            lo.erase(prev(lo.end()));
        }
        while (hi.size() > lo.size()) {
            lo.insert(*hi.begin());
            hi.erase(hi.begin());
        }

        // Window is full — record median
        if (i >= k - 1) {
            if (k % 2 == 1)
                result.push_back(*lo.rbegin());
            else
                result.push_back(((double)*lo.rbegin() + *hi.begin()) / 2.0);

            // Remove outgoing element
            int out = nums[i - k + 1];
            if (lo.count(out))
                lo.erase(lo.find(out));
            else
                hi.erase(hi.find(out));
        }
    }
    return result;
}
07

Exactly K Transform (Counting)

Counting Medium

Count subarrays with exactly K of something (distinct chars, odd numbers, etc.). Direct sliding window is hard because expanding can make the window valid, then invalid, then valid again.

The transformation: exactly(K) = atMost(K) − atMost(K−1)

Implement a helper countAtMost(K) using a standard variable-window where you contract when count > K. The number of valid subarrays ending at index right is simply right − left + 1.

💡 Why right − left + 1?

Subarrays ending at right that satisfy "at most K" start from any index in [left, right]. That's right − left + 1 subarrays.

▶ Subarrays with Exactly K Odd Numbers
C++ — Count Subarrays (Exactly K)
// Helper: count subarrays with at most K odd numbers
int atMost(vector<int>& nums, int k) {
    int left = 0, count = 0, result = 0;
    for (int right = 0; right < nums.size(); right++) {
        if (nums[right] % 2 == 1) count++;  // odd number found

        while (count > k) {
            if (nums[left] % 2 == 1) count--;
            left++;
        }

        result += right - left + 1;  // all subarrays ending at right
    }
    return result;
}

// exactly(K) = atMost(K) - atMost(K-1)
int subarraysWithExactlyKOdd(vector<int>& nums, int k) {
    return atMost(nums, k) - atMost(nums, k - 1);
}
08

Prefix Sums + Monotonic Deque

Hard Negative Numbers

When arrays contain negative numbers, the sum is no longer monotonic — adding an element can decrease the total. Standard sliding window fails.

The solution for "Shortest Subarray with Sum ≥ K" (LC 862) combines prefix sums with a monotonic deque:

Let P[i] be the prefix sum. We seek minimum j − i such that P[j] − P[i] ≥ K.

• Pop from front when P[j] − P[front] ≥ K (found valid, future j only gives longer)
• Pop from back when P[j] ≤ P[back] (j is a better starting point)

💡 Why Pop Back?

If P[j] ≤ P[back], then j is further right AND has a smaller or equal prefix sum — it's strictly a better candidate for future pairs.

▶ Shortest Subarray Sum ≥ K (LC 862)

Prefix Sums

C++ — Shortest Subarray Sum ≥ K
int shortestSubarray(vector<int>& nums, int k) {
    int n = nums.size();
    vector<long long> prefix(n + 1, 0);
    for (int i = 0; i < n; i++)
        prefix[i + 1] = prefix[i] + nums[i];

    deque<int> dq;  // indices with increasing prefix sums
    int minLen = INT_MAX;

    for (int j = 0; j <= n; j++) {
        // Pop front: found valid subarray
        while (!dq.empty() && prefix[j] - prefix[dq.front()] >= k) {
            minLen = min(minLen, j - dq.front());
            dq.pop_front();
        }

        // Pop back: j is a better starting point
        while (!dq.empty() && prefix[j] <= prefix[dq.back()])
            dq.pop_back();

        dq.push_back(j);
    }
    return minLen == INT_MAX ? -1 : minLen;
}
📚 C++ STL

STL Quick Reference

Essential C++ Standard Template Library containers and algorithms for sliding window problems.

std::vector

O(1) access
Fixed-size array processing, prefix sum storage. Random access in constant time.
📌 Use for: Fixed-size windows, prefix sums, result storage

std::deque

O(1) front/back
Double-ended queue with O(1) push/pop at both ends. The backbone of monotonic queue patterns.
📌 Use for: Sliding window max/min, monotonic queues

std::unordered_map

O(1) avg
Hash map for O(1) average insert, lookup, delete. Tracks character/element frequencies.
📌 Use for: Variable windows, frequency tracking, distinct elements

std::unordered_set

O(1) avg
Hash set for constant-time membership testing. Perfect for uniqueness constraints.
📌 Use for: Longest substring without repeating characters

std::multiset

O(log n)
Balanced BST allowing duplicates with ordered iteration. Supports rbegin() for max element.
📌 Use for: Sliding window median, dynamic sorted windows

std::priority_queue

O(log n)
Binary heap with O(log n) push/pop. Does not support arbitrary deletion — combine with lazy deletion.
📌 Use for: Top K in window, heap-based median

⚠️ Edge Cases & Guard Clauses

  • 🛡️ Length Validation: Check n ≥ k — if window size > input, return empty/error
  • 📍 Pointer Init: Both left and right start at index 0. Handle the first window correctly
  • 📏 Length Formula: Always use right - left + 1 for window size
  • 🗑️ Map Cleanup: Erase keys with 0 frequency from unordered_map to keep size() correct
  • 🔢 Off-by-One: Use i - k to find the outgoing element in fixed windows
  • 📊 Longest vs Shortest: Update result outside while loop (longest) vs inside (shortest)
  • ➖ Negative Numbers: If negatives are present, standard shrinking doesn't work — use prefix sums
